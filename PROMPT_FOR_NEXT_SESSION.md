### 一、 目标背景 (Background Context)
你好，我们目前已经完整开发并跑通了一个基于 Andrej Karpathy "Context Stuffing" 哲学思想的本地零向量（Zero Vector DB）知识库专家系统。
现有的核心架构（纯 Python 实现）：
- **Ingest**：提取各格式文件并规范化为纯文本，使用 SHA256 彻底去重，写入 `raw/` 目录。
- **Compile**：核心引擎，调用大语言模型作为编译器，阅读长文本并生成高密度摘要 (`xxx.summary.yaml`) 和细粒度业务结构本体 (`xxx.ontology.yaml`)，合并至 `wiki/index.yaml`。
- **Search**：精妙的三层渐进式降维检索（1. BM25 初筛 -> 2. LLM 精选候选集 -> 3. 原文注入长上下文进行答案推演并标注引用出处）。
- **Relate**：基于大模型的知识图谱动态扩充，扫描文档以树状网络固化文件之间的关联，保存在 `meta/relations/knowledge_graph.yaml` 中。
- **环境要求**：底座模型采用**阿里百炼 `qwen3.6-plus`** (兼容 OpenAI SDK 调用)，环境变量受 `.env` 隔离。核心架构逻辑在根目录严格遵循名为 `ANTIGRAVITY.md` 的 Schema 指令。

### 二、 当前核心任务 (Current Objectives)
现有的 CLI 极客版本已经通过端到端的实体测试（UAT）。在这个新的会话中，请你扮演资深全栈架构师，执行**从极客引擎向企业级知识库产品的整体演进**。你需要严格执行以下两个阶段规划：

#### 🚀 执行阶段一：后端服务化与异步重构 (FastAPI + Async)
1. **API 路由搭建**：引入 `FastAPI` 框架，在根目录创建 `app/` 或 `src/api/`，将底层的四个引擎脚本 (`ingest`, `compile`, `search`, `relate`) 封装抛出为标准的 RESTful 接口（例如：`/api/v1/search`, `/api/v1/upload`）。
2. **静默队列解耦**：由于 `compile` 环节重度依赖大模型的生成耗时极大，请使用 `BackgroundTasks`（或类似轻量化任务队列）将其与前台响应强解耦，做到"秒摄入，后台静默编译"。
3. **分词算法替换**：优化 Search.py Layer 1 的算法。在依赖中正式引入 `jieba` 中文分词库，替换掉现有的（只能依靠空格手工占位）的缺陷，让接口全面支持用户的长短句自然语言提问。

#### 🎨 执行阶段二：现代化企业前端基建 (React UI / Next.js)
1. **前端工程初始化**：在本项目根目录下新建 `frontend/` 目录，利用 Next.js (或 React + Vite) 脚手架进行初始化，接入 Tailwind CSS 或 Shadcn UI 来刻画极具科技感与优雅色彩的现代化体验界面。
2. **沉浸式 Wiki 知识库仪表盘**：前端读取后端的底层 Yaml 与 Index 统计，以卡片化形式构建全景知识节点，允许用户查阅当前编译状态。
3. **极客指令极速检索栏**：打造类 Mac `Raycast` 或者跨时代的悬浮大检索框。在问答界面支持调用 `/api/v1/search` 时启用 `Streaming` 响应处理流式逐字打字回复，并且一定要显式设计出"引用文献徽章 / Markdown 角标"。
4. **可视化 2D 关系图谱面板**：接入 `ECharts`、`React Flow` 或是类似的图形基建，把底层生成的 `knowledge_graph.yaml` 树状网转为非常漂亮的智能节点连接网可视化页面。鼠标悬浮可查看 AI 给出的生成理由。
