## 二、执行的具体改进

### 改进 1:仪表盘编译状态 bug 修复(M3,代码级)

**根因**:`compile.py` 只更新 `raw/{doc_id}.meta.yaml` 的 status,但 `wiki/index.yaml` 条目的 status 字段不同步(全滞留 None)。仪表盘读 index.yaml 统计 `status==='compiled'` → 恒为 0。

**修复**([api/main.py](api/main.py)):新增 `_enrich_doc_status()`,在 `/wiki/index` 返回时用 `.meta.yaml` 的权威 status 即时填充。
- ✅ 不改 compile.py 核心(守护栏)
- ✅ 只读合并,不写回 index.yaml(无副作用)
- ✅ 单篇 meta 读取失败不影响整体(降级)

**验证**:
- 单测 `test_wiki_index_enriches_status_from_meta` ✅(防回归)
- LIVE:后端 status 分布 `{compiled: 9}`(原全 None);浏览器仪表盘显示 `✓ compiled`

### 改进 2:用户手册 v2 重写(M1/2/4/5/6/8)
见 [docs/用户使用手册.md](docs/用户使用手册.md)。核心变化:
- 开头显著区块警告"首次检索慢"
- 5 分钟快速上手(4 步)
- 术语通俗化
- 故障排查表
- 结果质量判别表
- 每页补使用场景与价值

### 改进 3:回归验证
- 后端 pytest:**188 passed**(187 + 1 新守卫)
- 前端未改,基线保持(jest 81 / e2e 5 / build ✅)

## 三、重新规划(UX 驱动的路线图)

UX 评审改变了优先级判断——**体验瓶颈比功能扩展更影响落地**。调整后:

### 🔴 P0(落地阻塞,必须先做)

1. **检索性能优化**(原 P0,UX 评审后升为头号)
   - 现状:首次 30-60 秒,是用户流失的首要原因
   - 方案:jieba 预热(启动加载)+ Layer2 批量评分(多文档一次 LLM)+ 结果缓存
   - **这是落地的最大障碍**——再好的功能,用户等不到答案就没用

2. **Loop #4 合并**(操作侧已授权)
   - 前端接通 + 三份文档 + 本轮 bug 修复,一起合并

### 🟡 P1(落地后体验)

3. **提交方式优化**(走查新发现)
   - QA 页发送按钮不明显(走查时未找到标准发送按钮标签)
   - 方案:明确发送按钮 + Enter 提交提示

4. **检索进度反馈增强**
   - 现状:只有"正在检索..."文字
   - 方案:分步进度(粗筛中→精选中→生成中),让 30-60 秒等待不焦虑

5. **app/ 双后端同步**(原 P1)+ 稽核批量化(原 P1)

### 🟢 P2(锦上添花)
6. 截图补入手册(M7)+ lint 基线 + 文件格式扩展 + 跨文档实体 + 多用户权限

### 推荐的 Loop #5
**检索性能优化 + 进度反馈**(P0 #1 + P1 #4 合并)——这是落地与否的分水岭。
启动前需操作侧定价值标尺:"个人办公可用(P95<10s)" vs "试点部署(P95<5s)"。

## 四、落地准备度研判(诚实判定)

按"能否支撑实际应用落地"逐项评估:

| 维度 | 准备度 | 判定依据 |
|---|---|---|
| **功能完整度** | 🟢 高 | 存/查/懂/审四能力齐全 + LIVE 验证 + 188 测试 |
| **用户可见性** | 🟢 高 | Loop #4 接通 3 页面,真实用户可导航可操作 |
| **准确性** | 🟡 中 | 检索带引用可追溯;但 AI 可能出错,手册已警示需核对 |
| **性能** | 🔴 **不足** | 首次检索 30-60 秒——**这是落地的硬阻塞** |
| **稳定性** | 🟡 中 | 降级路径齐全;但未做长时间运行/并发压测 |
| **可运维性** | 🟡 中 | 单机启动 OK;无监控/日志聚合/健康检查端点 |
| **文档** | 🟢 高 | 手册 v2 + 路线图 + 评审报告齐全 |
| **安全性** | 🔴 未做 | 单机无认证;LLM Key 在 .env(已 gitignore);上线需补 RBAC |

### 总判定

> **更新(Big-Loop #5 后)**:性能硬阻塞已解除,个人办公场景从"有保留"升为"基本可用"。

**个人办公场景:基本可用** ✅
- ✅ 功能、可见性、文档达标
- ✅ **性能**:首检 14-21s + 重复 6s + 流式即时见字(Loop #5 实测,原 41s)
- ⚠️ AI 答案需人工核对(手册已警示)

**企业/多人场景:尚未就绪**
- ❌ 无认证/权限/审计(§0 红线 #3 安全模型,需操作侧授权专项)
- ❌ 无生产部署配置(集群/监控/备份)
- ❌ 性能未经并发验证

### 落地建议(分阶段)

1. ~~Loop #4 合并 + Loop #5 性能优化~~ → **已完成**(Loop #4 已合并 master,Loop #5 待合并)
2. **短期**:补健康检查端点 + 检索进度反馈 → 日常使用更顺滑
3. **中期(企业化专项)**:认证权限 + 生产部署 + 并发压测 → 团队/企业可用
