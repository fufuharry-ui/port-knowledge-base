## 🔴 P0 — 建议优先(价值高 / 成本中)

### 1. Loop #4 合并决策(阻塞中)
- **现状**:Loop #4(前端接通)已完成全闸门 + LIVE 实证,停在 dev 未合并
- **动作**:操作侧决定 merge 与否(§0 红线 #1,操作侧专属)
- **理由**:前端可视化是"用户可用"的闭环,不合并则 dev 与 master 分叉累积

### 2. 检索性能优化(用户体验硬伤)
- **现状**:首次检索 30–60 秒(jieba 冷启动 + Layer2 LLM 逐文档评分)
- **来源**:本次真实走查发现(非测试能抓到)
- **方案**:
  - jieba 预热(后端启动时加载一次)
  - Layer2 评分批量化(多文档一次 LLM 调用而非逐个)
  - 检索结果缓存(同查询短期复用)
- **价值**:检索是核心高频路径,慢 = 不可用。**这是当前最大体验瓶颈**

### 3. 上传编译状态可见性
- **现状**:仪表盘显示"已编译=0"、状态全 raw,但文档实际已有摘要/本体
- **来源**:走查发现(index.yaml 的 status 字段未正确回写 compiled)
- **方案**:排查 compile.py 的 status 回写 + 仪表盘统计逻辑
- **价值**:用户看不到"编译完成"会反复上传/困惑

---

## 🟡 P1 — 中期(价值中 / 成本中)

### 4. app/ 双后端同步(技术债)
- **现状**:`api/main.py`(前端用)有全部端点;`app/`(模块化)缺 `/entity-graph`、`/consistency`
- **来源**:Loop #4 评审登记。当前不影响用户(前端只连 api/main.py)
- **方案**:把两个端点移植到 `app/routers/`,保持双后端一致
- **触发**:一旦决定从 api/main.py 迁移到 app/,此债必须先还

### 5. 稽核增量/批量化(可规模化)
- **现状**:全库稽核 20 对 × 串行 LLM,文档量增大后成本/耗时线性增长
- **来源**:Loop #3 交付总结登记
- **方案**:增量稽核(仅对新入库文档跑候选对)+ 批量判定(多对一次 LLM)
- **价值**:文档量到 50+ 时,当前方式不可用

### 6. 跨文档实体关系抽取
- **现状**:实体关系仅在同一文档内抽取(term↔term 同源)
- **来源**:Loop #2 交付总结登记
- **方案**:扩展 extract_entity_relations 到跨文档实体对齐
- **价值**:更深的推理召回(如 A 文档的"5G专网"与 B 文档的"5G专网"关联)

## 🟢 P2 — 长期/增强(价值待验证)

### 7. lint 基线清理
- **现状**:`jest.setup.js`、`__mocks__/*.js` 的 CommonJS `require()` 被 eslint 误报 3 error
- **来源**:Loop #4 评审登记。新代码 0 error,不阻塞
- **方案**:eslint 配置加 overrides 排除 jest 基建文件
- **触发**:满足 DoD"0 lint error"目标时

### 8. 文件格式扩展(PDF/Word)
- **现状**:originals/ 仅接收 .txt
- **价值**:企业办公场景 PDF/Word 是主流
- **方案**:上传时加格式转换层(pdfplumber/python-docx → txt)

### 9. 实体图谱力导图可视化
- **现状**:实体关系用"邻居列表 + 关系边表格"(ADR-13,务实选择)
- **价值**:术语量大时图形化更直观
- **方案**:复用现有 KnowledgeGraph 组件(SVG 力导图)渲染实体图

### 10. 多用户/权限(企业级演进)
- **现状**:单机单用户,无 RBAC
- **价值**:从"个人"走向"企业办公"的必经路
- **方案**:引入认证 + 文档级权限 + 操作审计
- **注意**:属 §0 红线 #3 安全模型,需操作侧授权 + 两重证据 + 对抗评审

## 推荐的下一个 Big-Loop

**Loop #5 候选 = #2 检索性能优化**(P0 第 2 项)。

理由:
- 价值最高——检索是核心路径,当前慢到影响可用性
- 成本可控——主要是工程优化(jieba 预热 + 批量 LLM + 缓存),不涉及架构变更
- 可量化——有明确的 NFR 指标(P95 延迟),适合 Gate-3 ratchet
- 不碰红线——纯后端性能,不涉及数据模型/安全

但启动前需操作侧确认价值标尺(§2.1):例如"个人办公可用"(P95 < 10s)还是"试点部署"(P95 < 5s)。
