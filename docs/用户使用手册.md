## ⚠️ 开始前必读:第一次检索需等待(约 15–20 秒)

**这不是故障,是正常现象。** 系统会对候选文档让 AI 评分并生成带引用的回答。

你会看到页面显示 **"正在检索知识库..."**,随后答案会**逐字流式出现**(不用等全部生成)。
- 首次查询:约 **15–20 秒**到首字(之后答案流式生长)
- **重复同一查询:约 6 秒**(结果缓存)
- 若超过 60 秒仍无结果,见下方[故障排查](#六故障排查)。

> 性能说明:系统默认关闭 AI 的"深度思考"模式以加速(核心答案准确性已验证)。
> 如需更深推理,可在后端 `.env` 设 `ANSWER_ENABLE_THINKING=true`(更慢,约 30–40 秒)。

## 二、系统能为你做什么

一个中文知识库(港口智慧化领域)。它**不用向量数据库**,而是把文档编译成结构化的知识(摘要 + 概念分类 + 关系网),检索时把相关原文直接喂给 AI 回答。

四个能力:

| 能力 | 通俗说 | 对应页面 |
|---|---|---|
| **存** | 上传文档,自动摘要、抽概念、建关系 | ⬆️ 上传 / 📚 知识库 |
| **查** | 自然语言提问,得到带引用的答案 | 🔍 检索 / 🧠 问答 |
| **懂** | 看系统理解了哪些概念、概念间什么关系 | 🌳 本体 / 🔗 实体 / 🕸️ 图谱 |
| **审** | 检查多文档对同一事实有没有冲突 | 🛡️ 稽核 |

> **"三层检索"是什么意思?**(手册原版用了这个术语)就是系统找答案的三步:
> ① 关键词粗筛出 20 篇候选 → ② AI 精选最相关的 5 篇 → ③ 把这 5 篇全文喂给 AI 生成带引用的回答。
> 你不需要懂这个,只需知道:**答案有原文出处,可追溯**。

## 三、8 个页面怎么用(完整流程)

顶部导航 8 个入口:`📚 知识库 → ⬆️ 上传 → 🔍 检索 → 🧠 问答 → 🕸️ 图谱 → 🌳 本体 → 🔗 实体 → 🛡️ 稽核`

### 📚 知识库仪表盘(看家底)
看文档总数、已编译数、每篇文档的卡片(标题 + 摘要 + 日期 + 状态)。
- 状态含义:`✓ compiled`(已编译,可检索)/ `○ raw`(刚上传,编译中)/ `✗ error`(编译失败)
- **真实走查**:当前 9 篇文档,全部 compiled

### ⬆️ 上传(喂新文档)
拖拽 `.txt` 文件 → 自动入库 → 后台编译(摘要 + 概念 + 关系)。
- 上传后状态先 `raw`,编译完成变 `compiled`(刷新仪表盘可见)
- **注意**:目前只支持 `.txt`。PDF/Word 需先另存为纯文本
- 大文档编译可能要 1–2 分钟

### 🔍 检索(核心用法)
输入问题 → 点搜索(或 Ctrl+K 唤醒)→ 等待 → 看带引用的答案。
- 答案里的 `[文档标题 · 章节]` 是可追溯的原文出处
- **真实实证**:问"岸桥远控系统的网络延迟要求" → 答"视频回传≤50ms,控制信令≤20ms"+ 来源 doc_001/003

### 🧠 问答(带思考过程的检索)
和检索类似,但会展示**思考链**(检索→精选→生成每一步),答案逐字流式输出。
- 适合想了解"系统怎么得出结论"的场景
- 若命中文档间有已知矛盾,答案末尾会出现 `⚠️ 知识库内存在不一致` 提示

### 🕸️ 知识图谱(文档级关系网)
文档之间的关联网(同主题、补充、矛盾),力导图可视化。
- **什么时候用**:想横向看"哪些文档在讲同一件事"

### 🌳 本体知识树(系统理解的概念)
系统从所有文档抽取的概念分类树,可折叠。
- **真实实证**:92 个概念(智慧港口→港口自动化/数字化/安全…;人工智能→计算机视觉/大模型…)
- **什么时候用**:想了解知识库覆盖了哪些领域、某概念属于哪类
- **为什么有用**:这棵树会自动扩展你的搜索(搜"5G专网"会纳入上位"通信技术"和兄弟概念),提升找全率

### 🔗 实体关系图谱(术语级关系)
输入术语(如 `5G技术`)+ 跳数 → 看它的语义邻居(谁依赖谁、谁属于谁)。
- **真实实证**:`5G技术` 2 跳邻居:无人集卡、uRLLC、eMBB;eMBB `属于` 5G技术
- **什么时候用**:想搞清某个技术术语和相关术语的关系
- 点邻居标签可快速跳查

### 🛡️ 稽核(跨文档矛盾检查)
查看多文档对同一事实有没有冲突。
- 点「🔄 触发稽核」→ 系统对所有候选文档对跑 AI 判定(1–3 分钟)→ 看结果
- **真实实证**:当前 0 矛盾 / 20 候选对,显示"✅ 知识库内一致"
- **什么时候用**:文档来自不同来源/版本,担心同一指标说法不一致时

## 四、典型场景

**场景:刚接手一批港口文档,想快速了解"岸桥远控"**

1. 📚 仪表盘扫一眼有哪些文档
2. 🌳 本体定位"岸桥远控"在哪类,看它的上位/兄弟概念(建立全局观)
3. 🔍 检索问"岸桥远控的关键技术和网络要求"
4. 🔗 实体输入"岸桥远控",看它依赖什么技术、支撑什么场景
5. 🛡️ 稽核确认各文档对延迟/性能指标说法一致

## 五、怎么判断结果好不好(结果质量判别)

| 信号 | 含义 | 你的行动 |
|---|---|---|
| 答案带 `[标题 · 章节]` 引用 + 📎 来源 | 高质量,可追溯 | 直接用 |
| 答案有来源但含糊(如"相关文档提到…") | 中等,原文支持不足 | 点来源看原文核实 |
| `未找到相关文档` | 查询没命中 | 换关键词,或用本体/实体页找术语 |
| `⚠️ 知识库内存在不一致` | 文档间有冲突 | 两边都看,判断哪个更新/更权威 |
| 答案数字与原文不符 | AI 偶尔会错 | **务必核对引用原文**,别盲信 |

> ⚠️ **重要**:AI 生成的答案可能出错(尤其是具体数字)。**关键决策前,务必点开来源文档核对原文**。引用机制是为了让你能追溯,不是让你盲信。

## 六、故障排查

| 现象 | 可能原因 | 解决 |
|---|---|---|
| 页面打不开(localhost:3000) | 前端没启动 | 终端2 跑 `npm run dev` |
| 页面打开但显示"加载失败"/无数据 | 后端没启动 | 终端1 跑 uvicorn;确认 :8000 可访问 |
| 检索转圈超过 2 分钟 | LLM 服务慢/Key 问题 | 查后端终端有无报错;检查 `.env` 的 OPENAI_API_KEY |
| 上传后一直 `raw` 不变 `compiled` | 编译失败 | 查后端终端报错;可能是 LLM 调用失败 |
| 稽核按钮点了没反应 | 后端 LLM 慢 | 等待;查后端终端 |
| 本体/实体页空 | 数据未生成 | 确认文档已 compiled;跑 `python scripts/compile.py` |

**通用自检**:后端终端看到 `Uvicorn running on http://127.0.0.1:8000` + 前端看到 `Ready` = 系统健康。

## 七、已知边界

| 边界 | 说明 |
|---|---|
| 首次检索需等 | 15–20 秒到首字(见开头说明),之后流式生长;非故障 |
| 重复查询快 | 同查询 60s 内约 6 秒(缓存) |
| 稽核耗时 | 全库 20 对候选 × AI,约 1–3 分钟 |
| 矛盾>0 未实测 | 当前库无矛盾;矛盾列表渲染经自动化测试覆盖,但未在真实矛盾数据下走查 |
| 仅支持 .txt | PDF/Word 需先转文本 |
| 单机部署 | 本地 :8000 + :3000,非生产集群 |
| AI 可能出错 | 关键数字务必核对原文(见第五节) |
